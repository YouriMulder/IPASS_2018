Detailed Description

This class is used to communicate with the MFRC522 chip.

This is a basic library for the MF522 chip. Using the pure virtual class rfid. Execute the tests when you're not sure the chip is connected correctly.

Communicate with a potential rfid card.

Returns
An uint8_t containing a COMMUNICATION_STATUS value.
Parameters
commandthe command you want to execute.
transmitData[]the bytes you want to send to the rfid card. for instance: MIFARE_COMMAND
transmitLengththe amount of bytes you want to transmit to the card. receivedData[] a array to store the bytes you received from a potential rfid card.
receivedLengthThe amount of bytes you received from the rfid card.

A small summary of what happens in the method:

  1. The the register are set to start transmitting.
  2. The FIFO is reset.
  3. Command is executed.
  4. Checking for any errors.
  5. receivedData is filled with the FIFO data
    Warning
    You might have to change the BitFramingReg before using this command.
    THIS METHOD IS ONLY TESTED WITH Transceive COMMAND SO FAR!
    receivedLength MAY VARY.

◆ getCardUID()

uint8_t MFRC522::getCardUID ( uint8_t  UID[5])
overridevirtual

Get the UID/NUID of 4 bytes UID card with the BCC byte.

Parameters
UID[5]The array you want the UID to get stored into.
Returns
An uint8_t containing a possible error or succes code. (These codes are in the derived class)

The UID of a card is a unique identifier. This UID can be used to know which card you're communicating with. The UID is used when more than one card is in the radio frequency field, also known as collision detection.
The last byte of UID[5] will be the bcc. This can be used to check if the UID is valid.

Warning
Note that the UID might not be as unique as you might expect. Some cards have a feature to rewrite the UID block.
Note that the UID[5] array will be overridden.

Implements rfid.
